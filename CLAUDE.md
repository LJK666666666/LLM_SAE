## 代码编写要求

0.以下要求中如果存在不符合大语言模型领域主流习惯的设计，请指出并向用户确认
1.所有python代码执行路径尽量为项目根目录，比如执行 python src/main.py 而不是 cd src ; python main.py .
2.所有训练结果、模型权重、图表等等应当保存在 @results/{需要标注的实验设置}_{num} 文件夹下，其中{需要标注的实验设置}若已存在则{num}自动取最大值加1，避免覆盖之前的训练结果。恢复训练则接在原文件夹的训练记录后面继续保存。推理/评估/测试结果默认保存在训练结果的文件夹或子文件夹下，不要和训练结果分离开来保存。每次实验的配置信息和实验结果指标还应该统一保存到 @results/overall_config_metrics.csv 和 @results/overall_config_metrics.json 中。
3.如果训练神经网络，需要保存训练过程中每个epoch的学习率/训练集损失/验证集损失等详细信息（包括恢复训练所需要的所有信息），需要在每训练完1个epoch后立刻更新保存best和last两个模型权重（禁止在模型全部训练完成后才保存模型权重，避免训练中断导致模型权重丢失），需要支持根据last权重进行恢复训练。如果1个epoch的训练时间已经很长，尽量支持训练1个iteration（即只训练1个batch）方便快速测试。
4.所有图片中使用英文并且不得设置标题title。
9.使用中文跟用户交流。
11.对于可能需要较长运行时间的代码，应当添加进度条显示。
12.在编写.ipynb格式的notebook笔记本时，注意在添加了cell-{n}后，下一个代码单元应该添加在cell-{n+1}。如果不设置数字+1会导致后面的代码单元出现在前面的代码单元之前，从而出错。
13.如果需要创建markdown文件进行解释说明，请统一放置在'GUIDE'文件夹下。
14.如果你认为用户的命令不够合理或不符合主流代码编写习惯，请向用户提出并向用户确认。
15.为了代码库的可迁移性，所有路径应该设置为相对路径，禁止使用绝对路径。
16.如果编写神经网络代码且无任何参考时，建议将学习率调度机制和早停机制设置为每训练3个epoch验证集损失未下降，则学习率衰减为0.7；训练15个epoch学习率未下降，则早停。
17.高概率调整的重要参数放在命令行参数中，低概率调整的参数统一放在.yaml文件中。假如需要编写可视化脚本或数据分析脚本从 @results 结果文件夹中读取数据，建议编写一个.yaml文件设置读取的子文件夹名"{需要标注的实验设置}_{num}"，方便实验结果文件夹更新时进行快速调整。
18.构建一个文档树，互相引用以及引用其他非文档文件，管理全局全过程工作记忆，可随时查阅和快速回忆。专门创建 `_memory/` 文件夹存放全局性文档，也在关键子文件夹下创建 `memory.md`（模块化），混合使用。注意在 `_memory/` 文件夹以外的记忆文件统一使用 `memory.md` 文件名确保风格绝对独立于 `README.md` 且可以随时定制化。`_memory/` 文件夹里维护一个索引文件 `_memory/index.md`，列出所有 memory 文件的位置和简要说明，为散落的记忆文件提供统一入口。注意在代码发生新增、修改、删除时对 memory 文件进行同步更新。`_memory/` 文件夹里维护一个用户文件 `_memory/user.md`，记录所有想让用户注意或记住的信息，至少包括主要的运行命令行、在哪里修改一些重要配置/参数或开放性配置/参数、面对开放性问题下自己做出的决策等。
19.每次对话后在 'chat_history.md' 文件末尾添加记录用户提示词（可删减一些长段复制粘贴内容等）和对应工作内容的关键信息。如果记录文件较大，你想要读取时可以只读取末尾几条。
20.如果当前代码库与深度学习相关，可能需要迁移至colab云端运行训练推理，建议确保代码支持将结果保存到 /content/drive/MyDrive/results 或者 ../drive/MyDrive/results 路径下（至少支持一个）。
21.在情况允许（研究目标不在于batch_size的大小变化或影响不大）的条件下，尽量将训练/验证/测试的batch_size设置成：自动检测当前显存大小并设置成可行的最大batch_size
22.在调用大语言模型写提示词时，除非是严格的Role Playing场景，否则禁止使用“你是一个xxx领域专家”的提示词，不应当用一个身份让大语言模糊地判断生成内容的要求，而是将相关要求具体展开说清楚。
